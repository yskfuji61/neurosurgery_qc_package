# 5軸分離の原則

## 結論
薬剤や介入を評価するときは、以下の5軸を必ず分離する。

1. エビデンスの確実性
2. 推奨の強さ
3. 国内薬事上の位置づけ
4. 保険・査定リスク
5. 施設内運用可否

## 原資料要約
原資料は、B/C/D/X等の分類を残す場合も、それを医学的推奨度ではなく「施設内手順候補として施設内承認後に標準化できるかどうかを検討する」ものとして扱う方針を示している。

## GPTによる整理・解釈
RAGでは、5軸を同じchunkに混ぜず、JSONL上では `evidence_certainty`, `recommendation_strength`, `domestic_regulatory_status`, `insurance_risk`, `institutional_operability` を独立フィールドとして保持する。

## 誤読禁止
- エビデンスがあることを国内で標準運用できることと解釈しない。
- 海外GLにあることを国内適応内と解釈しない。
- 施設内運用可否を医学的推奨度と解釈しない。
- 診療支援候補を即時実装仕様と解釈しない。
